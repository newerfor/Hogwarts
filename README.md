Hogwarts Characters App 📱
🏰 Обзор приложения
Hogwarts Characters — мобильное приложение по вселенной Гарри Поттера с полной информацией о персонажах Хогвартса.

🎯 Функциональные возможности
👥 Управление персонажами
Полные профили с фото, датой рождения, факультетом и волшебными способностями

Два формата отображения: список (кратко) и детальные профили

Волшебные атрибуты: патронусы, волшебные палочки, происхождение

Статусы: жив/умер, студент/преподаватель, волшебник/магл

🏠 Факультетская система
Поддержка всех 4 факультетов (Гриффиндор, Слизерин, Пуффендуй, Когтевран)

Тематический дизайн с уникальными фонами и иконками для каждого факультета

Мгновенная идентификация принадлежности персонажа

🔍 Поиск и фильтрация
Поиск по имени с мгновенным результатом

Многоуровневая фильтрация:

По факультету

По должности (студент/преподаватель)

По волшебным способностям (волшебник/магл)

По происхождению (чистокровный/полукровка/маглорожденный)

Удобный диалоговый интерфейс для фильтров

📱 Пользовательский интерфейс
Адаптивная сетка персонажей под любой экран

Ленивая загрузка с пагинацией при прокрутке

Цветовые индикаторы статусов (жив/умер)

Плавные переходы между экранами

🛠 Технологическая реализация
🏗 Архитектура
Clean Architecture с разделением на Data/Domain/Presentation слои

MVVM с управлением состоянием через StateFlow

Repository Pattern для абсракции источников данных

Многомодульность — код разбит на независимые модули:

app — точка входа, навигация, DI

feature-main, feature-detail — экраны приложения

core-data, core-domain, core-ui, core-navigation — переиспользуемые библиотеки

🎯 Single Activity подход
В приложении используется единственная Activity (MainActivity)

Навигация между экранами через Jetpack Navigation Compose

Все экраны — Composable функции, управляемые NavHost

Чистый back stack без дублирования Activity

📦 Технологии
Kotlin — основной язык

Jetpack Compose — декларативный UI

Room — локальная БД с миграциями

Retrofit — работа с REST API

Koin — dependency injection

Coil — загрузка изображений

Material Design 3 — дизайн-система

🔧 Особенности реализации
Двухуровневое кэширование (память + диск)

Оптимизация изображений с проверкой дубликатов

Фоновая синхронизация без блокировки UI

Полная обработка ошибок с UI-состояниями (Loading/Success/Error)

Type converters для сложных типов данных

Пакетное сохранение в фоне

Мемоизация загруженных ресурсов
